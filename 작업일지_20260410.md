### 1. Admin 난입 기능 구현

**배경:** admin.html에서 AI가 고객과 대화 중인 세션에 관리자가 원격으로 난입, AI 대신 직접 대화할 수 있는 기능 요청

#### server.js 변경사항

- In-memory `sessions` Map 추가 — `Map<sessionId, { mode, messages, pendingAdminMsgs, ... }>`
- 비활성 세션 30분 후 자동 정리
- `SESSION_ID_RE` 정규식 검증 (`^S-\d{13}-[a-z0-9]{5}$`)
- `requireAdmin` 미들웨어 추가 — `Authorization: Bearer ADMIN_TOKEN` 검증
- CORS 와일드카드 → `onrender.com`, `localhost:3001`만 허용으로 제한

신규 API 엔드포인트 7개:

| 엔드포인트 | 설명 |
|---|---|
| `POST /api/session/register` | 고객 세션 등록 |
| `GET /api/session/status?id=` | 세션 상태 폴링 (고객 → 2초 주기) |
| `GET /api/admin/sessions` | 활성 세션 목록 조회 |
| `GET /api/admin/session/:id` | 특정 세션 메시지 조회 |
| `POST /api/admin/takeover` | admin 난입 (mode → 'admin') |
| `POST /api/admin/release` | AI에게 반환 (mode → 'ai') |
| `POST /api/admin/message` | admin → 고객 메시지 전송 |

`/api/chat` 수정:
- `sessionId` 파라미터 추가 → 세션 동기화
- admin 모드 시 `{ message: null, adminMode: true }` 반환 → AI 응답 차단

#### 방법 1 — IP Rate Limit (`express-rate-limit`)

```js
// 1분에 IP당 10회 초과 시 429 응답
const chatRateLimit = rateLimit({ windowMs: 60_000, max: 10 });
app.post('/api/chat', chatRateLimit, ...);
```

`package.json`에 `express-rate-limit: ^8.3.2` 추가

#### 1. 관리자 실시간 상담 개입 기능

기존에는 AI가 고객과 대화한 내용을 나중에 확인하는 것만 가능했습니다.

이제 **관리자 페이지에서 진행 중인 상담을 실시간으로 모니터링하다가, 필요한 시점에 직접 대화를 이어받을 수 있습니다.**

- 관리자 페이지에 "📡 라이브 상담" 탭 신규 추가
- 현재 진행 중인 상담 세션 목록 실시간 표시
- 세션 선택 시 대화 내용 실시간 확인
- "난입하기" 버튼으로 관리자가 직접 메시지 전송 가능
- "AI에게 넘기기" 버튼으로 AI 상담으로 복귀 가능

---

#### 2. 챗봇 어뷰징 방어

의도적으로 무관한 메시지를 대량 전송해 API 비용을 발생시키는 공격에 대응해 두 가지 방어 레이어를 적용했습니다.

- **속도 제한:** 동일 IP에서 1분에 10회 초과 시 자동 차단
- **내용 필터:** 드레스룸과 무관한 메시지는 저비용 모델이 먼저 판별 후 차단 → 메인 AI 호출 비용 절감

---

#### 3. 루마네 응답 말투 개선

AI 응답이 딱딱하게 느껴진다는 점을 개선했습니다.

- "안내드리겠습니다" → "알려드릴게요" 등 자연스러운 표현으로 통일
- 매 문장 "고객님" 반복 제거
- 한 번에 여러 질문 → 가장 중요한 것 하나씩 순서대로
