## 6. 핵심 파일별 현황

### server.js
- 시스템 프롬프트: AI임을 먼저 밝히지 않음, 물어볼 때만 솔직하게 답변
- 자기소개 형식 고정: "저는 케이트블랑 드레스룸 상담을 담당하고 있는 루마네입니다"
- `/api/chat` — 상담 응답 (claude-sonnet-4-6)
- `/api/summarize` — 상담 요약 + Supabase 저장 (claude-haiku)
- `/api/save-conversation` — 요약 없이 대화만 저장
- `/api/conversations` — 저장된 상담 목록 조회

### chat.html
- **PC (768px 이상)**: 좌측 보라색 사이드바(280px) + 우측 채팅 영역 (2컬럼 홈페이지 레이아웃)
- **모바일**: 풀스크린 채팅, 헤더 버튼 아이콘만 표시
- 헤더 `h-sub`: "케이트블랑 드레스룸 · AI 상담" (AI 표기법 준수, UI에만 표시)
- 채팅 메시지에는 AI 언급 없음
- 사이드바 버튼: 이전 상담이력, 수집현황, 상담저장
- `saveConversation()`: 모바일 버튼 + PC 사이드바 버튼 동기화 (`setSaveBtns` 헬퍼 사용)

### floorplan_preview.html
- SVG 도면 독립 페이지 (chat.html과 미연결)
- 축척 1:20 (1px = 20mm)
- 현재 하드코딩: ㄷ자 3000+2000+2100mm, 3단서랍장, 거울장
- 좌측벽: 560×3 + 3단서랍장760 + 560 = 3000mm
- 우측벽: 거울장600 + 500×3 = 2100mm
- 뒷벽 가시 중앙: 440×2 = 880mm

### 10_예외처리규칙.md
- 범위 외 질문 차단 규칙 추가됨
- 토큰 테러 방지: 날씨·요리·코딩·타브랜드 등 거절 후 상담 주제로 유도

## 7. Supabase DB 구조

테이블: `conversations`

| 컬럼 | 타입 | 설명 |
|------|------|------|
| id | int | 자동증가 PK |
| messages | jsonb | 대화 전체 배열 |
| summary | jsonb | Claude 분석 요약 (이름, 연락처, 치수 등) |
| created_at | timestamp | 생성시각 |

summary JSON 구조:
```json
{
  "이름": null,
  "연락처": null,
  "주소": null,
  "결제방식": null,
  "프레임색상": null,
  "선반색상": null,
  "천장커튼박스": false,
  "내용": null,
  "2단서랍": false,
  "3단서랍": false,
  "기둥추가": false,
  "5단선반": false,
  "배송비": null,
  "참고사항": null,
  "상담요약": "한 문장 요약"
}
```
