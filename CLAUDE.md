# 🤖 루마네 프로젝트 설정 (케이트블랑 드레스룸 AI 상담)

> 공통 작업 규칙은 전역 CLAUDE.md (`~/.claude/CLAUDE.md`) 에 있음.
> 본인 작업 스타일·환경 전용 설정은 `CLAUDE.local.md` (gitignore) 에 있음.

---

## 0. 핵심 운영 원칙

Claude Code는 **빠르게 작업하되, 위험한 수정은 반드시 멈추고 확인**한다.

### 기본 원칙
- 요청받은 부분만 수정한다.
- 임의 리팩토링, 임의 디자인 변경, 임의 문구 변경 금지.
- 불명확하면 추측해서 수정하지 말고 먼저 질문한다.
- `server.js`, Supabase/DB, 로그인, 저장, 삭제, 관리자 기능, 배포 관련 작업은 신중하게 처리한다.
- 큰 수정이 아니면 과도한 절차로 시간을 끌지 않는다.

---

## 1. 작업 시작 전 규칙

모든 작업을 시작할 때 매번 긴 보고서를 작성하지 않는다.  
작업 성격에 따라 **기본 작업**과 **확인 필요 작업**으로 나눈다.

### 1-1. 기본 작업 — 바로 진행 가능

아래 작업은 사용자 확인 없이 바로 진행해도 된다.

- 오타 수정
- 단순 문구 수정
- 작은 CSS/UI 조정
- 한 파일 안에서 끝나는 간단한 버그 수정
- 주석 정리
- console 로그 제거
- 명확하게 요청받은 작은 수정

단, 수정 후에는 간단히 보고한다.

```text
수정 파일:
변경 내용:
확인한 내용:
```

### 1-2. 확인 필요 작업 — 수정 전 반드시 멈춤

아래 작업은 파일을 수정하기 전에 먼저 사용자에게 확인받는다.

- 여러 파일을 동시에 수정하는 작업
- `server.js` 수정
- Supabase / DB 구조 변경
- 데이터 저장, 삭제, 수정 로직 변경
- 로그인 / 회원 / 권한 관련 수정
- 관리자 페이지 기능 수정
- 결제, 견적, 주문, 상담 저장 등 핵심 기능 수정
- 기존 디자인을 크게 바꾸는 작업
- 기능 구조를 바꾸는 리팩토링
- 배포 명령 실행
- 데이터 삭제 또는 초기화
- 사용자가 "먼저 확인해", "계획부터 말해", "어디 고칠지 먼저 알려줘"라고 말한 작업

이 경우 아래 형식으로 짧게 보고한다.

```text
요청 요약:
수정 예상 파일:
수정 범위:
건드리지 않을 부분:
백업 필요 여부:
진행해도 될까요?
```

사용자가 "진행"이라고 답하기 전까지 파일 수정 금지.

## 2. 백업 규칙

백업은 모든 작업마다 하지 않는다.  
아래 경우에만 백업한다.

- 여러 파일 수정
- 핵심 기능 수정
- `server.js` 수정
- Supabase/DB 관련 수정
- 관리자 기능 수정
- 저장/삭제/로그인 관련 수정
- 배포 전 큰 수정
- 사용자가 백업을 요청한 경우

백업 위치는 본인 PC 경로라 `CLAUDE.local.md` 참조.

백업 후에는 백업한 파일명만 간단히 보고한다.

---

## 3. 작업 완료 후 보고

작업 완료 후에는 길게 설명하지 말고 아래 형식으로 보고한다.

```text
완료했습니다.

요청 내용:
- 

원인 / 이유:
- 

해결 방법:
- 

결과: 성공 / 실패

수정 파일:
- 

추가 확인 필요:
- 없음 / 있음
```

실패한 경우에는 결과를 "실패"로 표시하고 이유를 간단히 적는다.

배포가 필요한 경우에는 반드시 별도로 말한다.

```text
배포가 필요합니다.
```

배포가 필요 없으면:

```text
배포 없이 로컬 수정만 완료했습니다.
```

## 4. 절대 금지사항

- 요청하지 않은 기능 추가 금지
- 요청하지 않은 디자인 변경 금지
- 기존 정상 기능 임의 수정 금지
- 한 파일에 모든 코드를 몰아넣기 금지
- 불명확한 요청을 추측해서 진행 금지
- DB 데이터 삭제 전 확인 없이 실행 금지
- 배포 명령 확인 없이 실행 금지
- `server.js` 허락 없이 수정 금지

## 5. 프로젝트 구조

### 주요 파일
- `chat.html` + `css/chat.css` + `js/` → AI 상담 채팅 앱 루마네
- `index.html`, `blog.html`, `quote.html` → 케이트블랑 홈페이지
- `admin.html` → 어드민 패널
- `server.js` → Express 서버, Anthropic API, Supabase, Notion 연동 **허락 없이 수정 금지**
- `preview_site/` → 작업 샌드박스, 배포 전 테스트용
- `백업본/` → 백업 저장소 (gitignore)

## 6. 루마네 AI 지침 폴더

서버가 시작될 때 `지침/` 폴더의 모든 `.md` 파일을 읽어 루마네 시스템 프롬프트에 자동 포함한다.

루마네 상담 관련 작업 시 아래 폴더를 먼저 확인한다.

```text
지침/
```

| 파일 | 내용 |
|------|------|
| `01_브랜드역할과상담톤.md` | 루마네 역할, 말투, 톤 |
| `02_배송비정책_최신.md` | 지역별 배송비 기준 |
| `03_설치치수계산규칙.md` | mm→cm 변환, 기본 행거 계산 |
| `04_할인규칙.md` | 신규아파트 10% 할인 조건 |
| `05_견적표원칙.md` | 견적서 출력 원칙 |
| `06_상담흐름.md` | 상담 진행 순서 |
| `07_필수확인항목.md` | 반드시 수집해야 할 고객 정보 |
| `08~16` | 응답 예시, 이미지 해석, 상담 스킬 등 |
| `18_옵션가격표.md` | 거울장·서랍장·선반 등 옵션 단가 전체 |

지침 파일 추가/수정 시:

```bash
git add 지침/파일명.md
git commit -m "지침 수정"
git push
```

## 7. 전체 작업 순서 요약

### 작은 작업
```text
요청 확인 → 바로 수정 → 간단 테스트 → 완료 보고
```

### 위험 작업
```text
요청 요약 → 수정 예상 파일 보고 → 사용자 확인 → 백업 → 수정 → 필요 시 검토 → 테스트 → 완료 보고
```

### 배포 작업
```text
수정 완료 → 테스트 → 사용자 확인 → 배포 → 배포 결과 보고
```
